# شرح مشروع Jordan Explorer 🇯🇴 (نسخة الصفحة الواحدة)

يا هلا فيك! بناءً على طلبك، جمعنا كل أكواد التطبيق في ملف واحد فقط وهو `lib/main.dart`. هاد الملف مجهز مخصوص عشان تفهم كل فتفوتة بالمشروع وكيف بيشتغل الكود من الصفر، حتى لو ما عندك أي خبرة سابقة بالبرمجة.

## 1. شو فكرة المشروع حالياً؟
بدل ما يكون عندنا ملفات كثيرة ومشتتة، حطينا كل شاشات التطبيق (الرئيسية، الفيديو، المعالم، الاختبار، والتقييم) جوا ملف واحد اسمه `main.dart`. 
بالبرمجة، بنقسم الشغل لمربعات أو طوبات (بنسميها Widgets)، وكل وحدة بتعمل وظيفة معينة. رح نمشي عليهم وحدة وحدة.

## 2. المصطلحات الأساسية اللي لازم تعرفها!
- **`Widget`**: أي اشي بتشوفه على الشاشة هو Widget. الزر، النص، الصورة، وحتى الشاشة كاملة تعتبر Widget.
- **`StatelessWidget`**: شاشة ثابتة (ما بتتغير). زي لوحة الإعلانات بالشارع، بتعرض كلام وصور بس ما بتتفاعل معك وما بتتغير ألوانها.
- **`StatefulWidget`**: شاشة متغيرة ومتفاعلة. زي التلفزيون أو اللعبة، كل ما تكبس زر بيتغير اشي عالشاشة (مثلاً لما تختار نجمة بالتقييم وتتلون).
- **`setState`**: هاي الكلمة السحرية اللي بنحكيها للتطبيق: "يا تطبيق، في اشي تغير! امسح الشاشة وارجع ارسمها بسرعة بالمعلومات الجديدة".

## 3. شرح كود `main.dart` بالتفصيل:

### أ. البداية وتجهيز التطبيق
أي برنامج فلاتر بيبدأ من نقطة معينة اسمها `main()`.

```dart
import 'package:flutter/material.dart'; 
import 'package:video_player/video_player.dart';

// الدالة main هي الباب الرئيسي. أول ما يشتغل التطبيق بيفوت من هون.
void main() {
  runApp(const JordanExplorerApp()); // بنحكيله: "شغّل التطبيق اللي اسمه JordanExplorerApp"
}

// هاد الكلاس هو الإعدادات العامة للتطبيق (اللبسة أو الثيم)
class JordanExplorerApp extends StatelessWidget {
...
    return MaterialApp( // الغلاف الرئيسي للإعدادات
      title: 'Jordan Explorer',
      theme: ThemeData( // هون بنحدد ألوان التطبيق (زي اللون البني الفاتح اللي بشبه رمل الصحراء)
        colorScheme: ColorScheme.fromSeed(
          seedColor: const Color(0xFFC19A6B),
        ),
...
      ),
      home: const MainScreen(), // // أول شاشة رح تطلع للمستخدم هي MainScreen
    );
}
```

### ب. الهيكل الرئيسي وشريط التنقل (`MainScreen`)
عشان نتنقل بين الأقسام (الرئيسية، المعالم، الخ...) بدون ما نفتح شاشات فوق بعض، عملنا شريط تحت (BottomNavigationBar) بيبدل المحتوى بالنص.

```dart
class MainScreen extends StatefulWidget { ... }

class _MainScreenState extends State<MainScreen> {
  int _currentIndex = 0; // هاد المتغير بيحفظ إحنا بأي صفحة حالياً. (0 يعني หน้า أولى، 1 يعني المعالم...)

  // قائمة (List) فيها الصفحات الأساسية اللي رح نبدل بينهم
  final List<Widget> _pages = [
    const HomeScreen(),       // رقم 0
    const AttractionsScreen(),// رقم 1
    const QuizScreen(),       // رقم 2
    const RatingScreen(),     // رقم 3
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold( // Scaffold هو الهيكل الأساسي (سقف، جسم، وأرضية)
      appBar: AppBar(title: const Text('Jordan Explorer')), // السقف
      
      body: _pages[_currentIndex], // الجسم: اعرض الصفحة بناءً على الرقم اللي اخترناه
      
      bottomNavigationBar: BottomNavigationBar( // الأرضية: شريط التنقل
        currentIndex: _currentIndex,
        onTap: (index) { // بس تكبس على أي أيقونة تحت...
          setState(() { // بنعمل تحديث للشاشة
            _currentIndex = index; // وبنغير الرقم للصفحة الجديدة!
          });
        },
        items: const [ ... ], // هون بنحط الأيقونات (البيت، القائمة، الاختبار، النجمة)
      ),
    );
  }
}
```

### ج. الشاشة الرئيسية (`HomeScreen`)
هاي الشاشة بتعرض الترحيب، لوجو التطبيق (اللي رسمناه بالأشكال الهندسية)، وزر مشاهدة الفيديو. شاشة ثابتة (StatelessWidget) لأنها بس بتعرض معلومات.
لما تكبس على زر الفيديو، بنستخدم كود الانتقال `Navigator.push` عشان تفتح شاشة الفيديو.

### د. شاشة الفيديو (`VideoScreen`)
هاي (StatefulWidget) لأننا بنحتاج نعرض فيديو بيتحرك وبنتحكم فيه (تشغيل/إيقاف).
استخدمنا مكتبة `video_player` الجاهزة.

```dart
// أول ما تفتح الشاشة بنشغل دالة initState عشان نجهز الفيديو من الملفات
_controller = VideoPlayerController.asset('assets/video/ved.mp4')..initialize();

// ولما نكبس على زر الـ Play بالنص:
onPressed: () {
  setState(() {
    if (_controller!.value.isPlaying) {
      _controller!.pause(); // اذا شغال وقفه
    } else {
      _controller!.play(); // اذا واقف شغله
    }
  });
}
```

### هـ. قائمة المعالم السياحية (`AttractionsScreen`)
بدل ما نكتب كود الأزرار 5 مرات، عملنا "قائمة بيانات" فيها (اسم المدينة، صورتها، ورمزها السري id).
وبعدين لفينا عليهم بطريقة ذكية وعملنا أزرار.
لما تكبس على أي زر، الكود بياخدك لـ `AttractionDetailsScreen` (شاشة التفاصيل) وبيبعتلها الـ `id` تبع المدينة اللي اخترتها عشان تعرف شو تعرض.

## الخلاصة للمبتدئ وببساطة:
لأننا جمعنا كل إشي بملف واحد، القراءة بتصير من فوق لتحت.
كل `class` بيمثل جزء أو صفحة بالتطبيق ومفصول عن التاني.
عشان يتواصلوا مع بعض:
- `MainScreen` هي المايسترو اللي بتبدل بين الصفحات الأساسية اللي تحتها كأنها قنوات تلفزيون.
- لما نحتاج نفتح صفحة جديدة بالكامل (زي الفيديو أو تفاصيل مدينة)، بنستخدم التاكسي `Navigator.push`.
- وكل مرة بدنا الشاشة تتغير بناءً على كبسة المستخدم، بنحط الشغل جوا (StatefulWidget) وبننادي التعويذة (setState).

وهيك كل الكود صار بملف واحد زي ما طلبت، والمشروع صار أسهل للفهم والتعديل!
